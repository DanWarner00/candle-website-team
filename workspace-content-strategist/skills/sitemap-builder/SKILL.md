---
name: sitemap-builder
description: Create a website sitemap and information architecture from a client brief and research findings. Trigger on "create sitemap", "plan the pages", "information architecture". Do not use for writing page copy.
---

1. Read `{project_dir}/client-brief.md` and `{project_dir}/content/research.md`.
2. Using the page list from the brief and research findings, define each page: slug, title, primary purpose (one sentence), key sections with brief descriptions (including each section's purpose and target word count), target user intent, primary SEO keywords.
3. Respect the content profile from research.md — visual-first sites need fewer, punchier sections. Information-first sites can have more detailed sections.
4. Write to `{project_dir}/content/sitemap.md`.
5. Announce completion with page count and names.
